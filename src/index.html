Description

The main page of NickBolles.com is just the start though. The server runs several NodeJS servers including:
  • NickBolles.com Users Server and MongoDB Database
  • FourSee Server and MongoDB Database
  • Private Git Server
  • And more to come...
The server is an Amazon EC2 Linux instance. I have setup and documented, the instance myself and automated many of the tasks related to setup and deploying updates to the server.

Future

  • I am continuing to improve the front end website every day, and update it with new information.
  • I am currently rolling out a new build and deploy process using Git, Git sub-modules, NodeJS and Forever along with Forever-Service.
    For More information Click Here

    New Deploy Process Details

    This new process makes heavy use of NodeJS for all servers and websites For example this website is run by a NodeJS static server on port 9000 (Try going to NickBolles.com:9000)
    When a request comes in on port 80 the server-router, a NodeJS Instance of http-master, routes the request based on the domain, sub-domain and path of the request to a certain port.

    • First, a Git "Server Repository" contains multiple Git sub-modules (for example nickbolles.com)
    • Second, running the "setup" script will install NodeJS (via nvm), and all npm modules needed. Then it will create a Forever-Service out of a "start-servers.js" file to keep the servers running as a service
    • When the "start-servers.js" Forever-Service starts, it will read all of the Git sub-modules, start them, and read their package.json for the routes that they require for the server-router
    • Then the "start-servers.js" script will create the http-master config file, allowing the server-router to dynamically allocate routes for the sub-modules currently included
    • After creating the config file for the server-router the "start-servers.js" script starts the router, and everything should be running and accessible!

    All in all it goes:
    1. Clone Git
    2. Run "Setup.sh"
    3. Access content
    Management of the service is easy with "sudo service nb-servers start|stop|restart|status"
